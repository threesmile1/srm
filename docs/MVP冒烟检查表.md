# MVP 冒烟检查表（可重复执行）

> 对应开发计划 **T8**：方案 [SRM建设方案.md](./SRM建设方案.md) **§3.2** 主链 + 常见异常。  
> 环境：后端 `8080`、管理端 `5173`、门户 `5174`；测试账号以种子数据为准（如 `admin` / `portal`）。

## 一、Happy Path（§3.2 七步）

| 步骤 | 操作要点 | 期望 |
|------|----------|------|
| 1 | 管理端维护供应商、物料、仓库（与采购组织一致） | 档案可查、无跨组织脏数据 |
| 2 | 采购订单：新建 → **提交审批** → 审批中心通过 → **发布** | 状态依次为待审批→已审核→已发布；门户可见 |
| 3 | 门户：订单行 **确认** 数量/交期 | `confirmedAt` 有值，待办带数字减少 |
| 4 | 门户：新建 **ASN** 关联已发布 PO | ASN 列表可见、管理端可按 PO 查看 |
| 5 | 管理端：**收货单** 勾 PO（及可选 ASN）、保存 | 行 `received_qty` 累计正确 |
| 6 | 管理端：**导出** PO / GR（U9 模板） | 文件可下载，`export_status` 更新 |
| 7 | 管理端：**执行报表** 选采购组织 | 数据与上述单据一致 |

## 二、异常与边界（抽样）

| 项 | 操作 | 期望 |
|----|------|------|
| 跨组织 PO | 行上仓库不属于订单采购组织 | 保存失败，提示明确 |
| 超收 | 收货量超过订单行可收+超收比例 | 拒绝或按配置提示 |
| 关单 | 已发布 PO **关闭** | 状态正确，门户不可再确认（按规则） |
| 门户越权 | 篡改 URL 中他人订单/ASN id | **403/404**（与 T4 一致） |
| 未登录调 API | 清除 Cookie 后直接 GET `/api/v1/purchase-orders` | **401**，JSON `error` 提示重新登录 |
| 审批 | PR/PO 已挂审批流时，单据页直批 | 提示走审批中心；中心 **一键通过** 带审批人 |

## 三、阶段 B 扩展（可选回归）

| 项 | 说明 |
|----|------|
| PR | 请购提交→审批→**转 PO** |
| 对账 | 按供应商+组织+期间生成对账单，金额与汇总一致 |
| 发票 | 门户提交→管理端确认/退回；**带订单行** 时：累计开票数量≤已收、单价与订单行一致（容差 0.01）、可选校验物料编码与收货单行数量 |
| 工作台 | 管理端首页卡片、待办跳转审批 |

## 四、自动化

- 后端：`srm-backend` 执行 `mvn test`（含门户 `MockMvc` 与 **401 无会话** 用例）。
- 全链路 E2E 可在后续接入 Playwright/Cypress，本表为手工与 CI 最小集对照。
